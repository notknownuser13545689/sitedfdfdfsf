Que's History

He had to do something so he made Que! The new generation Discord Bot for everyone to secure their Server Members from spam, scam and miss-information.

Que's Features

#1 AntiBot

Once enabled, will check if bot you invite to your server is Verified. If it's not Verified, Que will automatically kick the Unverified Bot.

Usage: q!antibot [on|off]

#2 AntiLink / AntiFlood

Once enabled, will check for links in all of the channels. If user sends a link, Que will delete User message and warn them.

Usage: q!antilink [on|off]

#3 Moderator Whitelist

Once enabled, will log every users actions. If the user is not in Whitelist and tries to change anything Moderation related, the actions will be reported to server owner.
Usage: q!whitelist add [user]

Que's FAQ

1. Why is Que banning other bots when adding to the server?

- It kicks bots because antibot system is enbaled. Disable it using this command: q!antibot off

2. Why Que bans bots/users who are helping with server customization, like editing/creating/delting channels/roles?

- It is because the user/bot who edit/create/delete channels/roles is not in whitelist. To add to whitelist, use this command: q!whitelist add @user

3. Que has a nuke/crash command? Some people say that it has a nuke command available only for developers.

- No, it does not have this command. A man called "СПЕКТР", had published the "real" code of Que, but it's fake.

4. The bot did not protect from nuke.

- There are some possible reasons why this could happen:
1. Bot was offline.
2. The Que Bot's role was lower than the nuker bot/user.
3. Nuker bot/user was in whitelist.

5. Make sure you protect your server as much as possible!

1. Put the Que role on the most top you can.
2. Enable antibot - q!antibot on
3. Enable antiflood and anti-link - q!antiflood on & q!anti-link on
